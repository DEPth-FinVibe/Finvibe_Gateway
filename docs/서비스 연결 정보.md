# 서비스 연결 정보
- 이 프로젝트는 Spring Cloud Gateway를 사용하여 API 게이트웨이 역할을 합니다.

## JWT 인증
모든 요청에 대해 JWT 토큰을 사용해서 인증을 처리하여야 합니다.

## 라우팅 정보
- 아래 Caddyfile 정보를 참고
```caddyfile
# 시장 웹소켓 요청은 /api prefix만 제거해 Docker nginx로 전달
@market_ws path /api/market/ws /api/market/ws/*
handle @market_ws {
        uri strip_prefix /api
        reverse_proxy 127.0.0.1:18090
}

# /api 로 시작하는 요청은 WAS로 전달
handle_path /api/* {
        reverse_proxy 127.0.0.1:8080
}
```
