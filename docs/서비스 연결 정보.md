# 서비스 연결 정보
- 이 프로젝트는 Spring Cloud Gateway를 사용하여 API 게이트웨이 역할을 합니다.

## JWT 인증
JWT 인증 상세 내용은 `docs/JWT 인증.md` 문서를 참고합니다.

## TokenFamily 상태 조회
- Gateway는 보호 경로 요청 시 Redis에서 TokenFamily 상태를 직접 조회합니다.
- 기본 key 형식은 `token-family:{familyId}` 입니다.
- Redis hash에는 최소 `status`, `expiresAt` 필드가 있어야 합니다.
- AccessToken에 `token_family_id` claim이 없으면 Gateway가 요청을 차단합니다.
- `/api/auth/**`는 공개 경로를 유지하며, refresh token family 검사는 인증 서비스가 직접 처리합니다.

## 라우팅 정보
- 아래 Caddyfile 정보를 참고
```caddyfile
# 시장 웹소켓 요청은 /api prefix만 제거해 Docker nginx로 전달
@market_ws path /api/market/ws /api/market/ws/*
handle @market_ws {
        uri strip_prefix /api
        reverse_proxy 127.0.0.1:18090
}

# /api 로 시작하는 요청은 WAS로 전달
handle_path /api/* {
        reverse_proxy 127.0.0.1:8080
}
```
